It's possible to set items to any specific slot in the inventory, to the offhand and to the armour slots. The mod can detect which players have never joined before. If another mod adds items, such as guide books, they can either be removed completely or added after the kit has been set. This is configurable via the <em>addExistingItemsAfterKitSet</em> config option.<br><br>Can be used in a server for all new players or in singleplayer to start off with a default inventory of your choosing. There is also optional support for FTB's Team Island mod after the create island command.<br><br>The mod can be used <strong>server-side only</strong>. If you want players to be able to use the "Choose Kit GUI", it has to be installed on the client as well. Alternatively, they can choose kits via server commands.<br><br><span style="font-size:24px"><strong>Features:</strong></span><br>&nbsp;&bull;&nbsp; A guide here in the description. Difficulty ranging from <strong>Simple</strong>, <strong>Intermediate</strong> and <strong>Advanced</strong>.<br>&nbsp;&bull;&nbsp; Give players joining the world for the first time custom starter items and/or potion effects.<br>&nbsp;&bull;&nbsp; Support for multiple kits. Can be given randomly or chosen by the player.<br>&nbsp;&bull;&nbsp; There is custom tracking to determine who are new players, compatible with existing worlds.<br>&nbsp;&bull;&nbsp; Lots of commands! For creating, adding, managing and choosing starter kits.<br>&nbsp;&bull;&nbsp; If installed on the client, kits can be chosen via a custom GUI screen. If not, commands can be used.<br>&nbsp;&bull;&nbsp; Add descriptions to your kits. RPG modpack, anyone?<br>&nbsp;&bull;&nbsp; Remove other modded starter items, such as guide books. Or add them after the starter kit.</span></p><p style="text-align:center"><br><br><img src="https://cdn.serilum.com/description/nameplate/dependencies.svg" alt="Dependencies" width="838" height="64"></p><p><a href="https://curseforge.com/minecraft/mc-mods/collective" target="_blank" rel="nofollow"><img src="https://cdn.serilum.com/description/badges/dependencies/requires/collective.svg" alt="Requires Collective" width="251" height="49"></a><br><a href="https://curseforge.com/minecraft/modpacks/the-vanilla-experience" target="_blank" rel="nofollow"><img src="https://cdn.serilum.com/description/badges/dependencies/includes/the-vanilla-experience.svg" alt="Included in The Vanilla Experience" width="350" height="49"></a><a href="https://curseforge.com/minecraft/mc-mods/serilums-customization-bundle" target="_blank" rel="nofollow"><img src="https://cdn.serilum.com/description/badges/dependencies/includes/customization-bundle.svg" alt="Included in Serilum's Customization Bundle" width="416" height="49"></a><br><img src="https://cdn.serilum.com/description/badges/environment/server.svg" alt="Environment: Server" width="220" height="49"></p><p style="text-align:center"><br><br><br><img src="https://cdn.serilum.com/description/nameplate/configurable.svg" alt="Configurable" width="838" height="64"></p><p><a href="https://serilum.com/configuration-guide" target="_blank" rel="nofollow"><img src="https://cdn.serilum.com/description/badges/configuration-guide.svg" alt="Configuration guide" width="220" height="49"></a></p><p>&nbsp;&nbsp;&nbsp;<span style="font-size:18px">All available config options:</span></p><div class="spoiler" style="margin-left:10px;padding-left:10px"><p><span style="font-size:14px"><strong>randomizeMultipleKitsToggle</strong></span><span style="font-size:12px;color:#8a8d91">&nbsp;&nbsp;default true</span><br><span style="font-size:13px">When multiple starter kits are added via /starterkit add, there are two ways to distribute them. With this enabled, one is chosen at random. When disabled, players can choose one on first join.</span><br><br><span style="font-size:14px"><strong>addExistingItemsAfterKitSet</strong></span><span style="font-size:12px;color:#8a8d91">&nbsp;&nbsp;default true</span><br><span style="font-size:13px">Whether items that existed in the inventory, such as books added by other mods, should be added back to the inventory after the kit was set. If disabled, they'll be removed. You can still manually set them via the kit.</span><br><br><span style="font-size:14px"><strong>usePotionEffectsInStarterKit</strong></span><span style="font-size:12px;color:#8a8d91">&nbsp;&nbsp;default true</span><br><span style="font-size:13px">If potion/mob effect functionality should be enabled. This means that when creating a kit via /sk (add/set), it also saves the active effects the player has. And when handing out the starter kits, it adds the effects to new players.</span><br><br><span style="font-size:14px"><strong>formatKitNames</strong></span><span style="font-size:12px;color:#8a8d91">&nbsp;&nbsp;default true</span><br><span style="font-size:13px">If kit names should be formatted. Each word will be capitalized.</span><br><br><span style="font-size:14px"><strong>chooseKitScreenHeader</strong></span><span style="font-size:12px;color:#8a8d91">&nbsp;&nbsp;default "%s, you can choose a starter kit!"</span><br><span style="font-size:13px">The text used above the kit choice screen, or if the mod is not installed in the chat. %s will be replaced with the player's name.</span><br><br><span style="font-size:14px"><strong>announcePlayerKitChoiceInDedicatedServer</strong></span><span style="font-size:12px;color:#8a8d91">&nbsp;&nbsp;default true</span><br><span style="font-size:13px">Whether an announcement should be broadcasted to the server whenever a new player makes a kit choice. 'randomizeMultipleKitsToggle' must be disabled, and there must be at least 2 starter kits available.</span><br><br><span style="font-size:14px"><strong>enableFTBIslandCreateCompatibility</strong></span><span style="font-size:12px;color:#8a8d91">&nbsp;&nbsp;default true</span><br><span style="font-size:13px">Whether the starter kit should be re-set after the '/ftbteamislands create' command from FTB Team Islands. Does nothing when it's not installed.</span></p></div><p style="text-align:center"><br><br><br><img src="https://cdn.serilum.com/description/nameplate/commands.svg" alt="Commands" width="838" height="64"></p><p><span style="font-size:14px"><br><em>Both '/starterkit' and '/sk' can be used.</em><br><br><strong><span style="text-decoration:underline">Admin Commands:</span></strong></span><br><span style="font-size:14px"><span style="font-size:14px">/sk add (kit_name)</span> - <em>Adds your current inventory to the active kits.</em></span><br><span style="font-size:14px"><span style="font-size:14px">/sk set (kit_name)</span> - <em>Sets your current inventory as the only active kit.</em></span></p><p><span style="font-size:14px"><span style="font-size:14px">/sk give &lt;player&gt; (kit_name)</span> - <em>Give a player a random (or specific) starter kit.</em></span></p><p><span style="font-size:14px"><span style="font-size:14px">/sk activate &lt;kit_name&gt;/_all</span> - <em>Moves kit_name or all kits from inactive to active.</em></span><br><span style="font-size:14px"><span style="font-size:14px">/sk deactivate &lt;kit_name&gt;/_all</span> - <em>Moves kit_name or all kits from active to inactive.</em></span></p><p><span style="font-size:14px"><span style="font-size:14px">/sk list (inactive)</span> - <em>Lists all active (or inactive) kits.</em></span><br><span style="font-size:14px"><span style="font-size:14px">/sk reset</span> - <em>Resets all tracking. Players receive another kit on join.</em></span><br><span style="font-size:14px"><span style="font-size:14px">/sk reload</span> - <em>Reloads starter kit changes from the config folder.</em></span><br><br></p><p><span style="text-decoration:underline;font-size:14px"><strong>User Commands:</strong></span><br><span style="font-size:14px"><span style="font-size:14px">/sk choose &lt;kit_name&gt;</span> - <em>Allows players to choose a starter kit via commands. This is only needed when Starter Kit is not installed on the client and the server enabled kit choices.</em></span><br><span style="font-size:14px"><span style="font-size:14px">/sk info &lt;kit_name&gt;</span> - <em>Shows the description and items of kit_name.</em></span><br><span style="font-size:14px">/sk help</span> -<em> Shows the Starter Kit commands in-game.</em></p><p style="text-align:center"><br><br><br><img src="https://cdn.serilum.com/description/nameplate/description.svg" alt="Description" width="838" height="64"></p><p><span style="font-size:36px"><strong>Usage Guide:</strong></span><br><span style="font-size:14px">The mod can be configured in many different ways. To keep a better overview, I've divided the difficulty into three different sections. <strong>Simple</strong>, <strong>Intermediate</strong> and <strong>Advanced</strong>.</span><br><br><span style="font-size:18px"><strong>Simple:</strong></span><br><span style="font-size:14px">Set a custom starter kit. Add/Remove starter items from other mods. And giving players a starter kit via commands.</span><br><br></p><p><span style="font-size:18px"><strong>Intermediate:</strong></span><br><span style="font-size:14px">Add an extra starter kit. Deactivate a kit via commands. Add a starter kit with potion effects, and setting the level and duration.</span><br><br></p><p><span style="font-size:18px"><strong>Advanced:</strong></span><br><span style="font-size:14px">Make players choose a kit on first join. Move the inactive default kits to active. Show the choose kit GUI screen. Create a starter kit with a custom description.</span><br><br></p><p><span style="font-size:36px"><strong>Section 1: <span style="font-size:36px">Simple</span></strong></span><br><span style="font-size:14px">This is the default starter kit, which will be given when the mod is installed without any configuration:</span><br><img src="https://cdn.serilum.com/projects/starter-kit/0.png" alt=""><br><br></p><p><span style="font-size:14px">You can set your own starter kit by using the '<em><strong>/starterkit set (kit_name)</strong></em>' (or '<em><strong>/sk set (kit_name)</strong></em>') command. The kit name is optional.</span><br><span style="font-size:14px">After using this command, all other active starter kits are moved to the inactive folder.</span><br><img src="https://cdn.serilum.com/projects/starter-kit/1.png" alt=""><br><br></p><p><span style="font-size:14px">Here you can see the new starter kit data file that was created with the previous command. It's located in '<em><strong>./config/starterkit/kits/</strong></em>':</span><br><img src="https://cdn.serilum.com/projects/starter-kit/2.png" alt=""><br><br></p><p><span style="font-size:14px">That's it! All new players will now receive the "<strong>Upgraded</strong>" kit when they first join the world:</span><br><img src="https://cdn.serilum.com/projects/starter-kit/3.png" alt=""><br><br></p><p><span style="font-size:14px">Inside the config file, located at '<em><strong>./config/starterkit.json5</strong></em>', you can choose whether or not other modded starter items should be removed or given after the kit.</span><br><span style="font-size:14px">Examples of these modded starter items are guide books.</span><br><img src="https://cdn.serilum.com/projects/starter-kit/4.png" alt=""><br><br></p><p><span style="font-size:14px">To give other players (or yourself) a starter kit manually, you can use the '<em><strong>/starterkit give &lt;player&gt; (kit_name)</strong></em>' command.</span><br><span style="font-size:14px">The kit_name argument is optional, if left out:</span><br><span style="font-size:14px">If the mod is configured via intermediate, a kit may be given randomly.</span><br><span style="font-size:14px">If the mod is configured via advanced, the kit choice screen may be shown.</span><br><img src="https://cdn.serilum.com/projects/starter-kit/5.png" alt=""><br><br></p><p><span style="font-size:36px"><strong>Section 2: <span style="font-size:36px">Intermediate</span></strong></span><br><span style="font-size:14px">You can add another active kit by using the '<em><strong>/starterkit add (kit_name)</strong></em>' command.</span><br><span style="font-size:14px">With the default configuration, this means that new players will now randomly receive the "<strong>Default</strong>" or "<strong>No Bread</strong>" starter kit.</span><br><img src="https://cdn.serilum.com/projects/starter-kit/6.png" alt=""><br><br></p><p><span style="font-size:14px">To see all active starter kits, use the '<em><strong>/starterkit list</strong></em>' command. To see inactive starter kits, use '<em><strong>/starterkit list inactive</strong></em>':</span><br><img src="https://cdn.serilum.com/projects/starter-kit/7.png" alt=""><br><br></p><p><span style="font-size:14px">To disable an active kit, use '<em><strong>/starterkit deactivate &lt;kit_name&gt;</strong></em>'. This moves it to the '<em><strong>./config/starterkit/kits/inactive/</strong></em>' folder.</span><br><img src="https://cdn.serilum.com/projects/starter-kit/8.png" alt=""><br><br></p><p><span style="font-size:14px">The mod can also give potion effects alongside the starter kit! Make sure '<strong>usePotionEffectsInStarterKit</strong>' is enabled in the config.</span><br><span style="font-size:14px">Then, just make sure to have the potion effect enabled, and use the '<em><strong>/starterkit set</strong></em>' or '<em><strong>/starterkit add</strong></em>' command.</span><br><span style="font-size:14px">The effect duration will be set to what the player has exactly now, but we will round that up in a later step.</span><br><img src="https://cdn.serilum.com/projects/starter-kit/9.png" alt=""><br><br></p><p><span style="font-size:14px">For this example, I've given the player the speed potion effect. This was active when using the previous command:</span><br><img src="https://cdn.serilum.com/projects/starter-kit/10.png" alt=""><br><br></p><p><span style="font-size:14px">Locate and open the kit we've just created. The path is '<em><strong>./config/starterkit/kits/Speed_Kit.txt</strong></em>':</span><br><img src="https://cdn.serilum.com/projects/starter-kit/11_1.png" alt=""><br><br></p><p><span style="font-size:14px">This is the data file for the "<strong>Speed Kit</strong>" kit. It contains the items the player will receive and at the bottom what effects will be given to the player.</span><br><span style="font-size:14px">When a kit is created without active effects, that line will be empty. Otherwise it copies the exact level and duration.</span><br><span style="font-size:14px">In case of this example, it contains a reference to the Speed effect. It's <strong>lvl 1</strong> with a duration of <strong>3341 ticks</strong>, which equals 167 seconds. <strong>20 ticks = 1 second</strong>.</span><br><img src="https://cdn.serilum.com/projects/starter-kit/11_2.png" alt=""><br><br></p><p><span style="font-size:14px">I'm going to change the effect a player will receive to <strong>Speed level 5</strong>, with a duration of <strong>5 minutes</strong>. 6000 ticks = 300 seconds = 5 minutes:</span><br><img src="https://cdn.serilum.com/projects/starter-kit/12.png" alt=""><br><br></p><p><span style="font-size:14px">You can reload any changes you've made to files inside '<em><strong>./config/starterkit/</strong></em>' via the '<em><strong>/starterkit reload</strong></em>' command:</span><br><img src="https://cdn.serilum.com/projects/starter-kit/13.png" alt=""><br><br><span style="font-size:14px">That's it! Whenever a player first joins a world, they will now receive the "<strong>Speed Kit</strong>" kit. Which contains a few standard items and the <strong>Speed V</strong> effect, with a duration of <strong>5 minutes</strong>:</span></p><div class="spoiler"><p><img src="https://cdn.serilum.com/projects/starter-kit/14.webp" alt=""></p></div><p>&nbsp;<br><br><span style="font-size:36px"><strong>Section 3: <span style="font-size:36px">Advanced</span></strong></span><br><span style="font-size:14px">Starter Kit also contains a custom kit choosing screen!
